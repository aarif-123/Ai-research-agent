LangGraph state object — typed schema TypeScript
// AgentState — flows through every node, append-only
{
  company: string,               // "Apple Inc." or "AAPL"
  ticker: string,                // resolved by Planner
  plan: string[],                // 6 sub-questions
  toolResults: ToolResult[],   // { tool, data, error? }
  evidence: {
    scores: Record<Factor, number>, // 7 factors 0-100
    risks: Risk[],              // { label, severity, desc }
    news: NewsItem[],           // { headline, tone, source }
    keyMetrics: KeyMetrics      // price, PE, revenue, margin
  },
  verdict: {
    decision: 'INVEST'|'PASS'|'HOLD',
    score: number,               // 0-100 confidence
    rationale: string,          // full paragraph
    positionSizing: string     // e.g. "moderate, hedge China"
  },
  sessionId: string              // SSE stream key
}
Prompt engineering — per node Critical
NodeOutput formatKey prompt directives
Planner JSON string[] Resolve ticker. Generate exactly 6 research questions covering: (1) financials, (2) growth narrative, (3) recent news, (4) SEC risk disclosures, (5) competitive position, (6) macro/sector context. Return only JSON array.
Research Tool calls Tool dispatcher only — no LLM call. Use Promise.allSettled over all 8 tool functions. Partial failures are acceptable; log them as warnings in state.
Synthesis Strict JSON You are a senior equity analyst. Given raw tool data for {ticker}, produce a structured evidence object. Score each of 7 factors 0–100 with brief justification. Extract top 4 risks with LOW/MEDIUM/HIGH severity. Extract top 4 news items with tone float −1 to +1. Return only valid JSON matching the Evidence schema. Do not add commentary outside the JSON.
Verdict Strict JSON You are a portfolio manager at a top-tier fund. Given the evidence object, decide: INVEST if score ≥ 65 and no HIGH-severity risks dominate; HOLD if 45–64 or mixed signals; PASS if score < 45 or structural red flags. Provide a confidence score 0–100, a 3–4 sentence rationale citing specific metrics, and a position sizing note. Return only JSON. Never hedge the verdict.
Key decisions and trade-offs README section
Chosen — and why
SSE over WebSockets: unidirectional, Vercel-native, zero extra infra
LangGraph StateGraph over LangChain chain: explicit node boundaries, clean SSE emit points, conditional retry edges
Yahoo Finance (free) as primary data: no API key friction for evaluators, sufficient for MVP scope
Separate Synthesis + Verdict nodes: lets you A/B prompt the decision logic independently of data aggregation
Promise.allSettled for tools: partial data beats full failure; graceful degradation
Strict JSON output format on Synthesis/Verdict: predictable parsing, no hallucinated prose leaking into structured fields
Left out — would add with more time
Redis caching of tool results (same ticker re-fetches unnecessarily)
Historical backtesting: compare agent verdicts against actual 1-year returns
PDF ingestion of annual reports (Tier 2 — needs PDF parsing + chunking)
Rate-limit handling on free-tier APIs with exponential backoff
Auth layer — currently single-user, no session isolation
Comparable companies deep-dive: full peer-group financial comparison table
